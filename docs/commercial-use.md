# Commercial Use

RepoMori is source-available software, not open-source software. The public
license allows personal and non-commercial use under the PolyForm
Noncommercial License 1.0.0. Commercial use requires a separate written license
from TWO HANDS NETWORK LTD.

This page is a practical guide, not a substitute for the license text. The
binding terms are in [LICENSE.md](../LICENSE.md), [NOTICE.md](../NOTICE.md),
and [COMMERCIAL-LICENSE.md](../COMMERCIAL-LICENSE.md).

## Usually Allowed Under The Public License

These uses are generally intended to fit the public non-commercial grant:

- personal experimentation and learning;
- hobby projects without commercial purpose;
- non-commercial research, evaluation, and education;
- reading the source to understand how RepoMori works;
- filing issues or contributing under [CONTRIBUTING.md](../CONTRIBUTING.md).

## Requires Written Commercial Permission

Ask before using RepoMori for:

- a paid product, hosted service, managed service, or enterprise workflow;
- commercial AI coding, agent, memory, context, retrieval, or repository-analysis
  products;
- commercial model training, fine-tuning, distillation, benchmarking,
  evaluation, or product improvement;
- bundling, redistributing, or operating RepoMori for customers;
- embedding RepoMori code, schemas, tests, pack format details, or derived
  implementation details into a commercial tool;
- private support, warranty, procurement, indemnity, or service-level terms.

No commercial license is granted unless agreed in writing by TWO HANDS NETWORK
LTD.

## Evaluation By A Company

If a company wants to evaluate RepoMori, keep the evaluation narrow:

- use isolated test repositories or non-sensitive source;
- keep generated `.repomori` packs, handoffs, and memory timelines private;
- do not use RepoMori output to improve a commercial model or product unless a
  written license allows it;
- contact TWO HANDS NETWORK LTD before production, customer-facing, or
  revenue-supporting use.

## What To Include In A Commercial Request

Include:

- company and product name;
- intended use case;
- whether RepoMori would be embedded, hosted, redistributed, or used internally;
- expected users, customers, or systems;
- whether generated packs or outputs would cross organization boundaries;
- requested support, warranty, procurement, security, or compliance terms.

## Enterprise Review Notes

RepoMori is designed to be local-only and model-free by default. It does not
require provider API keys for its core commands. That helps privacy and
procurement review, but it does not make generated artifacts safe to publish.
Treat packs and handoffs as source-bearing artifacts.

See [security-model.md](security-model.md) and
[enterprise-readiness.md](enterprise-readiness.md).
