# RepoMori 0.2.0 Final Promotion Checklist

Use this after `0.2.0rc1` has had enough public validation time and no blocking
issues remain.

## Blockers

Do not promote if any of these are true:

- GitHub `tests` is failing on `main`.
- GitHub `release-candidate` is failing for the latest RC tag.
- A published wheel install smoke fails outside the checkout.
- `release-check --fail-on low` reports active findings.
- License, notice, commercial-use, or ownership wording needs legal changes.
- A user-facing command or schema contract needs a breaking fix.

## Version And Notes

- Change `pyproject.toml` from `0.2.0rc1` to `0.2.0`.
- Change `CHANGELOG.md` heading from `0.2.0rc1 - 2026-06-18` to the final
  `0.2.0` release date.
- Add `docs/releases/0.2.0.md` based on the RC notes and validation findings.
- Keep `docs/releases/0.2.0rc1-validation.md` as historical proof.

## Local Gates

Run from a clean checkout:

```powershell
python -m unittest discover -s tests

python -m repomori release-check D:\Dev\RepoMori `
  --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json `
  --fail-on low `
  --artifacts-dir D:\Dev\RepoMori\.repomori-release-candidate\final-release-check `
  --drift-log D:\Dev\RepoMori\.repomori-release-candidate\final-release-check\baseline-drift.jsonl `
  --json

python -m repomori memory D:\Dev\RepoMori `
  --out-dir D:\Dev\RepoMori\.repomori-packs `
  --no-handoff `
  --json

python -m repomori release-health D:\Dev\RepoMori `
  --snapshot-dir D:\Dev\RepoMori\.repomori-packs `
  --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json `
  --artifacts-dir D:\Dev\RepoMori\.repomori-release-candidate\final-health `
  --drift-log D:\Dev\RepoMori\.repomori-release-candidate\final-health\baseline-drift.jsonl `
  --json
```

## Package Smoke

Build a final wheel locally, install it into a fresh virtual environment, then
run:

```powershell
repomori schema --json
repomori demo --out D:\Dev\RepoMori\.repomori-release-candidate\final-demo --force --json
repomori contract-check --fixture D:\Dev\RepoMori\tests\fixtures\compat-contracts.json --json
```

## Tag And Release

Only after local and GitHub gates pass:

```powershell
git status --short
git tag -a v0.2.0 -m "RepoMori 0.2.0"
git push origin v0.2.0
```

Then create the final GitHub release:

```powershell
gh release create v0.2.0 `
  --repo Martin123132/RepoMori `
  --title "RepoMori 0.2.0" `
  --notes-file D:\Dev\RepoMori\docs\releases\0.2.0.md
```

## After Release

- Confirm the release page has the wheel and source archive attached.
- Confirm the final tag `tests` and release-candidate/package workflow are green.
- Update any external README snippets or project description if the release notes
  exposed wording drift.
