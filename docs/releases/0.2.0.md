# RepoMori 0.2.0

RepoMori `0.2.0` is the first public source-available release line.

## Highlights

- Offline `.repomori` repository packs with compressed source chunks, hashes,
  file metadata, imports, symbols, headings, summaries, and exact source
  recovery.
- Agent-facing `context`, `capsule`, `handoff`, `diff-context`, `eval`, and
  `context-eval` commands.
- Snapshot memory loop with `doctor`, safe `prune`, `timeline`, tamper-evident
  chain verification, anchors, and `release-health` checks.
- Dependency-free JSON-lines `agent` bridge and MCP stdio bridge.
- Public-release safety scan, baseline drift telemetry, compatibility contracts,
  and CI artifact bundles.
- Source-available licensing under PolyForm Noncommercial terms, with
  commercial use reserved for separate written permission.

## Validation Basis

This final release is promoted from `0.2.0rc1`.

The RC was validated from the published GitHub wheel in a fresh virtual
environment outside the checkout. See
[`0.2.0rc1-validation.md`](0.2.0rc1-validation.md) for the full validation
record.

The final release asset was also validated after publication. See
[`0.2.0-validation.md`](0.2.0-validation.md).

Final promotion gates:

- `python -m unittest discover -s tests`
- `python -m repomori release-check D:\Dev\RepoMori --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json --fail-on low --json`
- `python -m repomori release-health D:\Dev\RepoMori --snapshot-dir D:\Dev\RepoMori\.repomori-packs --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json --json`
- final wheel install smoke from a fresh virtual environment
- GitHub `tests` workflow on `main`
- GitHub release package workflow on tag `v0.2.0`

## Notes

RepoMori remains local-only and model-free. It does not call OpenAI, Claude, or
any provider API, and it does not require API keys.
