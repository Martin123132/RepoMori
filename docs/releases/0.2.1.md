# RepoMori 0.2.1

Maintenance release notes for the `0.2.1` release.

Status: final release notes prepared. The GitHub release should be published
only after the maintainer reviews the refreshed draft assets and validation
record.

Draft release validation is recorded in
[`0.2.1-validation.md`](0.2.1-validation.md).

## Focus

`0.2.1` is a polish and trust release, not a pack-format migration.
It keeps RepoMori local-only, dependency-free for the core runtime path, and
compatible with existing `repomori.pack.v1` packs.

This release proves:

- the company-wide non-commercial license posture is visible and checked;
- public release artifacts carry checksum, provenance, SBOM, evidence, and
  policy-verification material;
- memory anchors, release-health, and workflow contract checks stay green;
- first testers can follow one short install/demo/context/handoff route;
- generated packs, reports, rehearsals, and logs stay under `D:\...` or hidden
  `.repomori-*` paths.

## Highlights

- License policy gate covering TWO HANDS NETWORK LTD ownership, personal and
  non-commercial public use, written commercial licensing, and COO commercial
  contact wording.
- Release integrity bundle with `checksums.txt`, `release-provenance.json`,
  `sbom.spdx.json`, `verify-release`, release evidence, and policy profiles.
- Draft-first `publish-release` workflow that verifies assets before a
  maintainer publishes the GitHub release.
- Release rehearsal and privacy guard checks for reviewer-facing artifacts.
- First tester checklist for public install, demo, context, handoff, and storage
  assumptions.
- CI hardening for release-health, memory-anchor, workflow contracts, and
  generated-artifact preflights.

## Local Validation Before Release

Run from a clean checkout:

```powershell
python -m unittest discover -s tests
python -m repomori release-check D:\Dev\RepoMori --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json --fail-on low --json
python -m repomori release-health D:\Dev\RepoMori --snapshot-dir D:\Dev\RepoMori\.repomori-packs --baseline D:\Dev\RepoMori\.repomori-scan-baseline.json --json
python -m repomori release-rehearsal --out D:\Dev\RepoMori\.repomori-release-rehearsal --force --json
```

Before publishing `0.2.1`, confirm the draft release assets and validation
record have been refreshed from the final source commit.

## Reviewer Notes

This release does not add model/provider integration, network calls, encryption,
or a UI. It is deliberately a stabilization step so the existing offline memory,
handoff, MCP, release-health, and release-integrity surfaces feel coherent
before larger product work resumes.
